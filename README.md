# Systems Programming Project - SIC Two-Pass Assembler

This project is a Python implementation of a SIC assembler with a complete two-pass workflow:

- Pass 1 builds the symbol table and intermediate representation.
- Pass 2 generates object code records and listing output.

It is designed for systems programming coursework and demonstrates core assembler concepts such as location counters, symbol resolution, machine opcode translation, and object program formatting.

## What The Project Does

Given a SIC assembly source file, the assembler will:

1. Parse source statements (with optional labels and operands).
2. Validate opcodes and assembler directives.
3. Compute addresses using LOCCTR.
4. Build SYMTAB and detect duplicate symbols.
5. Write an intermediate file from Pass 1.
6. Optionally run Pass 2 to produce:
   - object program file (H/T/E records)
   - listing file with address, source fields, and object code

## Core Modules

- `main.py`
  - Command-line entry point.
  - Runs Pass 1 by default.
  - Runs Pass 2 when `-pass2` flag is provided.

- `core/pass1.py`
  - Reads and parses SIC source lines.
  - Handles comments (`.` or `;`) and blank lines.
  - Supports directives: `START`, `END`, `WORD`, `RESW`, `RESB`, `BYTE`.
  - Maintains `SYMTAB`, `LOCCTR`, and program metadata.
  - Writes intermediate output to `output/intermediate_files/`.
  - Prints symbol table and program length.

- `core/pass2.py`
  - Reads intermediate file generated by Pass 1.
  - Resolves symbol addresses from `SYMTAB`.
  - Generates machine/object code for SIC instructions and data directives.
  - Produces:
    - object file in `output/object_code_files/`
    - listing file in `output/listing_files/`

- `op_codes/op_table.py`
  - SIC opcode table used by both passes.
  - Stores opcode hex values and whether an operand is required.

## Command-Line Usage

Run from the project root.

### Pass 1 only

```bash
python main.py -filename samples/sample_code_1.txt -output sample.mdt
```

This produces:

- `output/intermediate_files/sample.mdt`
- terminal SYMTAB table
- terminal program length

### Pass 1 + Pass 2

```bash
python main.py -filename samples/sample_code_1.txt -output sample.mdt -pass2
```

This additionally produces:

- `output/object_code_files/sample.obj`
- `output/listing_files/sample.lst`

## Input Format

Accepted statement shapes:

- `LABEL OPCODE OPERAND`
- `OPCODE OPERAND`
- `OPCODE`

Comments can be full-line or trailing and start with `.` or `;`.

Example:

```asm
COPY      START      1000
FIRST     STL        RETADR
          LDA        LENGTH
          RSUB
EOF       BYTE       C'EOF'
LENGTH    WORD       3
BUFFER    RESB       4096
          END        FIRST
```

## Output Files

### Intermediate file (`.mdt`)

- Contains line-by-line addresses plus cleaned source statements.
- Used as input for Pass 2.

### Object file (`.obj`)

- Uses SIC object program record format:
  - Header record: `H`
  - Text records: `T` (max 30 bytes each)
  - End record: `E`

### Listing file (`.lst`)

- Shows address, label, opcode, operand, and generated object code per line.
- Useful for debugging and verification.

## Supported Directives And Encoding Rules

- `START`: sets program start address.
- `END`: marks end of source and execution start symbol.
- `WORD`: generates 3-byte object code.
- `BYTE`:
  - `C'...'` encodes characters as ASCII hex.
  - `X'...'` encodes raw hexadecimal bytes.
- `RESW` and `RESB`: reserve memory (no object code emitted).

## Error Cases Detected

- Duplicate symbols in Pass 1.
- Invalid opcode/directive.
- Undefined symbol references in Pass 2.
- Missing operands where required.
- Invalid `BYTE` operand format.

## Sample Run

```bash
python main.py -filename samples/sample_code_2.txt -output sample2.mdt -pass2
```

Then inspect:

- `output/intermediate_files/sample2.mdt`

## Notes

- Parsing is whitespace-based.
- The project uses in-memory globals in Pass 1 (`SYMTAB`, `LOCCTR`, etc.) for a single run lifecycle.
- Output directories must exist (already included in this project structure).
